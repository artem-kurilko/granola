# Granola House - Інтернет-магазин натуральної граноли

Ласкаво просимо до Granola House - інтернет-магазину натуральної граноли домашнього приготування. Наш магазин пропонує різноманітні смаки граноли, виготовленої з якісних інгредієнтів без консервантів та штучних добавок.

## Опис проекту

Це повноцінний інтернет-магазин з наступним функціоналом:
- Каталог продуктів з детальним описом
- Кошик для покупок
- Система оформлення замовлень
- Інтеграція з LiqPay для онлайн-оплати
- Сторінки публічної оферти та політики конфіденційності
- Адаптивний дизайн для всіх пристроїв
- **База даних для зберігання даних користувачів, замовлень та продуктів**

## Технології

- React + TypeScript
- Vite (швидка збірка)
- Tailwind CSS (стилізація)
- shadcn/ui компоненти
- React Router (навігація)
- TanStack Query (робота з API)
- **Supabase (хмарна база даних PostgreSQL)**

## Робота з базою даних

### Як працює база даних

Проект використовує **Supabase** як хмарну базу даних. Supabase - це відкрите альтернатива Firebase, яка надає:
- Базу даних PostgreSQL
- Аутентифікацію користувачів
- REST API для взаємодії з даними
- Безкоштовний план для стартапів

### Де зберігаються дані

Всі дані зберігаються в хмарі Supabase. Конкретно:
- Дані про користувачів (ім'я, email, телефон)
- Замовлення та історія покупок
- Інформація про продукти
- Відгуки клієнтів

Дані доступні через API Supabase, яке ми використовуємо в клієнтській частині додатка.

### Як підключити Supabase

1. **Створіть акаунт на Supabase**
   - Перейдіть на [supabase.com](https://supabase.com)
   - Зареєструйтеся та створіть новий проект

2. **Знаходьте ваші ключі**
   - У вашому проекті Supabase перейдіть в "Settings" → "API"
   - Запишіть:
     - `Project URL` (це ваш `VITE_SUPABASE_URL`)
     - `anon` public key (це ваш `VITE_SUPABASE_ANON_KEY`)

3. **Створіть .env файл**
   - Скопіюйте файл `.env.example` в `.env`:
     ```bash
     cp .env.example .env
     ```
   - Замініть значення в `.env` на ваші ключі Supabase:
     ```
     VITE_SUPABASE_URL=your_supabase_url
     VITE_SUPABASE_ANON_KEY=your_supabase_anon_key
     ```

4. **Налаштуйте таблиці в базі даних**
   - У Supabase перейдіть у "Table Editor"
   - Створіть наступні таблиці:
     - `products` (id, name, description, price, image_url, created_at)
     - `orders` (id, user_id, total, status, created_at)
     - `order_items` (id, order_id, product_id, quantity, price)
     - `users` (id, name, email, phone, address, created_at)

5. **Налаштуйте політики доступу**
   - Для кожної таблиці налаштуйте правила доступу через "Row Level Security"

### Приклад використання Supabase в коді

```typescript
// src/lib/supabase.ts
import { createClient } from '@supabase/supabase-js'

const supabaseUrl = import.meta.env.VITE_SUPABASE_URL
const supabaseAnonKey = import.meta.env.VITE_SUPABASE_ANON_KEY

export const supabase = createClient(supabaseUrl, supabaseAnonKey)
```

Приклад отримання продуктів з бази даних:

```typescript
import { supabase } from '@/lib/supabase'

const fetchProducts = async () => {
  const { data, error } = await supabase
    .from('products')
    .select('*')
  
  if (error) {
    console.error('Error fetching products:', error)
    return []
  }
  
  return data
}
```

### Переваги використання Supabase

- **Безкоштовний для стартапів** - Supabase надає безкоштовний план з достатньо ресурсів для малих проектів
- **Відкрите джерело** - Ви можете запускати Supabase локально або власному сервері
- **Простий API** - Зручний інтеграційний інтерфейс для React
- **Аутентифікація** - Вбудована система аутентифікації користувачів
- **Хмарне сховище** - Для зберігання зображень та інших файлів

## Встановлення та запуск

### Передумови

Переконайтесь, що у вас встановлено:
- Node.js (версія 16 або вище)
- npm або yarn

### Кроки для запуску

1. Клонуйте репозиторій:
```bash
git clone <url-репозиторію>
cd granola-house
```

2. Встановіть залежності:
```bash
npm install
```

3. Створіть файл `.env` з вашими ключами (див. розділ "Як підключити Supabase")

4. Запустіть сервер розробки:
```bash
npm run dev
```

5. Відкрийте в браузері: http://localhost:8080

### Доступні команди

- `npm run dev` - запуск сервера розробки
- `npm run build` - збірка продакшн версії
- `npm run preview` - перегляд зібраної версії локально
- `npm run lint` - перевірка коду лінтером

## Розробка

1. Додайте новий продукт через панель управління Supabase
